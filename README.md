\# 🍅 \[최종 기획서] 생산성 향상을 위한 적응형 게이밍 타이머: 다마도로 (Damadoro) 개발 명세서



\---



\## 1. 프로젝트 개요



\* 

\*\*프로젝트명:\*\* 다마도로 (Damadoro - 다마고치와 뽀모도로의 합성어) 





\* 

\*\*기획 배경:\*\* 뽀모도로 타이머에 게임화(Gamification) 요소와 육성 시뮬레이션 요소를 결합하여 사용자의 장기적인 집중과 몰입을 유도함.





\* 

\*\*핵심 가치:\*\* 작업 상태와 사용자 행동에 유동적으로 반응하는 '적응형 인터페이스(Adaptive interface)'의 구축.





\* 

\*\*개발 환경 가이드:\*\* 기획 데이터와 핵심 비즈니스 로직을 온전히 보존하여 CLI 및 로컬 개발 환경에서 최고의 정확도로 구현할 수 있도록 설계함.







\---



\## 2. 핵심 타이머 및 보상 프로세스



본 시스템은 뽀모도로 기법의 기본 원칙인 집중과 휴식의 순환 구조를 충족하면서, 세션 연속 달성에 강력한 명분과 동기부여를 제공하도록 설계되었습니다.



\* 

\*\*기본 세션 관리 (Session Management):\*\* 25분의 몰입 집중(Focus) 세션과 5분의 단기 휴식(Break) 세션을 1세트로 묶어 관리합니다.





\* 

\*\*연속 성공 보너스:\*\* 사용자가 중단 없이 세션을 연속으로 성공할 때마다 획득 재화에 점진적인 가산 배율이 적용됩니다.





\* 

\*\*장기 휴식 (Long Break) 규칙:\*\* 총 4번의 세션(4세트)을 완주하면 15분\~30분의 긴 휴식 세션으로 진화합니다.





\* 

\*\*적응형 휴식 스킵 제한 로직 (Adaptive Feature):\*\* 긴 휴식 돌입 시, 초기 15분 동안은 '휴식 스킵 버튼'이 강제로 비활성화됩니다. 15분이 경과한 시점부터 버튼이 활성화되어 남은 시간을 건너뛸 수 있게 만듦으로써 사용자의 완전한 휴식을 강제성 있게 보장합니다.

\## 3. 가챠 및 아이템 수집(도감) 시스템



복잡한 장착 UI 구현 리소스를 제거하는 동시에 사용자의 수집욕을 극대화할 수 있도록 '소지 기반 도감제'와 '합성 메커니즘'을 이원화하여 구축했습니다.



\* 

\*\*직관적인 아이템 네이밍 스키마:\*\* 모든 아이템의 명칭 접두사에 시너지 대상 캐릭터를 명시하여 추가 설명 없이 직관적으로 매칭을 이해하도록 유도합니다.





\* 

\*예시:\* `\[안경 방울토마토]의 스마트 안경` 









\* 

\*\*도감 완성 보상 (패시브 효과):\*\* 가챠를 통해 새로운 아이템을 획득하여 도감을 채울 때마다 전체 재화(토마토 잎사귀) 획득량에 영구적인 패시브 보너스 배율이 누적 적용됩니다. 중복이나 저등급 아이템 획득 시의 박탈감을 상쇄하는 장치입니다.





\* 

\*\*아이템 장착 보상 (액티브 효과):\*\* 캐릭터별로 제공되는 단일 슬롯에 아이템을 장착하면 해당 캐릭터의 '호감도' 상승량에 추가 배율이 제공됩니다. 이때 다마고치와 찰떡궁합인 전용 아이템을 매칭하면 추가 시너지 배율이 폭발적으로 가산됩니다.





\* 

\*\*5-in-1 승급 및 합성 (Merge) 시스템:\*\* 동일한 하위 등급 아이템 5개를 소모하여 확정적으로 바로 윗 단계 등급 아이템 1개를 합성해 낼 수 있습니다. 이는 무한한 성장의 동력이자 별도의 가챠 천장이 필요 없는 안전장치 역할을 수행합니다.

\## 5. 확장형 데이터 아키텍처 및 자동화 설계



컨텐츠 확장(캐릭터 및 아이템 개수의 증가) 시 메인 소스코드를 일절 수정하지 않고 데이터 파퓰레이션만으로 연동되도록 마스터-세이브 구조를 완전 분리합니다.



\### A. 마스터 데이터 (Master Data) 구조



\* 

\*\*다마고치 마스터 테이블 (`characters`):\*\* 고유 식별 코드(`id`), 명칭, 컬러 등 기본 정보 관리.





\* 

\*\*아이템 마스터 테이블 (`items`):\*\* 시너지 대상을 역추적할 수 있는 `targetCharacterId` 속성을 포함시켜 도감 연동과 네이밍 조합을 자동화함.







\### B. 유저 세이브 데이터 (User Save Data) 구조



\* 소지 재화, 현재 선택 캐릭터, 각 다마고치별 호감도 스탯 수치를 맵(Map) 구조로 보존합니다.





\* 

\*\*인벤토리 최적화:\*\* 소지한 아이템의 `\[ID: 개수]` 쌍만 딕셔너리 형태로 가볍게 관리하여 인벤토리가 무한히 확장되어도 용량 증가를 억제합니다.







C. 등급별 편의성 및 자동화 해금 시스템 



인벤토리에 특정 등급 이상의 아이템이 보존되는 것만으로 타이머 인터페이스가 유동적으로 변형되며 편의 기능이 자동 적용됩니다.



\* 

\*\*A등급 (영웅) 보유 시 - \[🖐️ 자동 쓰다듬기]:\*\* 5분 휴식 시간에 유저가 직접 상호작용하지 않아도 타이머 작동 중 주기적으로 소량의 호감도가 자동 가산됩니다.





\* 

\*\*S등급 (전설) 보유 시 - \[💧 자동 물 \& 밥 주기]:\*\* 밥과 물을 수동으로 챙겨야 하는 번거로움을 제거하여 유저가 25분 집중에만 오롯이 몰입하도록 환경을 제어합니다.





\* 

\*\*UR등급 (신화) 보유 시 - \[🛡️ 호감도 하락 영구 방지]:\*\* 육성의 최대 감점 요인인 시간 경과 및 탈주에 따른 애정도 감소 수치를 원천 차단하는 종결 보너스입니다.

\## 6. 사운드 및 알림 경험 기획 (Audio UX)



사용자가 화면을 지속적으로 인지하지 않아도 청각을 통해 현재 세션 상태를 명확히 구별하고 행동 리듬을 바꿀 수 있도록 디자인합니다.



\* \*\*세션 완료 청각 피드백:\*\*

\* 

\*집중 종료 알람:\* 맑고 청량한 종소리 계열의 사운드와 함께 재화가 정산되는 경쾌한 효과음을 연출하여 도파민 분비를 촉진합니다.





\* 

\*휴식 종료 알람:\* 리듬감 있고 차분한 나팔 또는 뻐꾸기 소리를 배치하여 깜짝 놀라지 않으면서 자연스럽게 다시 몰입으로 복귀하도록 유도합니다.









\* \*\*인게임 상호작용 SFX 명세:\*\*

\* 

\*볼따구 인터랙션:\* 마우스 드래그 인장 시 쫀득하고 찌이익 늘어나는 질감의 소리 제공. 마우스를 놓는 즉시 탄성 진동의 주파수와 감쇠 속도에 완벽히 동기화되어 소멸하는 `\[띠용- 부르르릉-]` 사운드 연출.





\* 

\*가챠 및 합성 완료:\* 열매를 수확하듯 가볍게 뽁 뽑히는 소리 및 연산 완료 시 단단하게 무기가 제련된 듯한 타격감 있는 쾌감 사운드 맵핑.









\* 

\*\*환경 적응형 백색소음 (Adaptive White Noise):\*\* 집중 세션 중 아날로그 시계 초침 소리를 기본 제공하며 , 고등급 아이템 수집 상태에 따라 캐릭터 콘셉트와 어우러지는 전용 환경음(예: 풋토마토-시골 마을 빗소리, 흑토마토-심야 카페 소음)이 지능적으로 자동 전환되어 몰입감을 제고합니다.

\## 7. 데이터 저장 및 예외 처리 명세



로컬 및 CLI 환경에서 유저가 의도적으로 프로세스를 강제 종료하거나 데이터를 되돌려 어뷰징하는 행위를 완전 격리하기 위한 트랜잭션 및 세이브 트리거 명세입니다.



\### A. 실시간 데이터 저장 시점 (Save Triggers)



1\. 

\*\*가챠 상점 호출 즉시 (Anti-Reroll):\*\* 가챠 실행 버튼을 누르는 순간, 화면 연출이 출력되기 전에 \*\*재화 차감과 뽑기 결과 아이템 확정 연산이 끝난 직후\*\* 즉시 로컬 데이터를 기록합니다. 저등급 아이템 획득 시 강제 종료하여 재화를 복구하는 리세마라 꼼수를 원천 차단합니다.





2\. 

\*\*합성 및 승급 연산 완료 즉시:\*\* 소모 아이템 5개 차감과 상위 아이템 1개 가산 연산이 교차하는 즉시 세이브를 수행하여 데이터 유실 및 재료 증발 버그를 예방합니다.





3\. 

\*\*세션 및 패널티 발생 즉시:\*\* 25분 집중 완수, 휴식 완수, 중도 포기 확인창 클릭 즉시 스탯 변동 정보를 실시간으로 파일에 안전하게 보존합니다.
